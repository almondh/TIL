## DATAHERE의 인공지능 기반의 실시간 맞춤 상품 추천 솔루션 정리
### 1. Sales Funnel
- 시장 내 상품 공급자간 경쟁 심화 -> 최종 구매까지 각 단계별 전환 과정 길고 복잡
- 신규 트래픽 유입 증대보다 유입고객의 구매 전환율 확보 중요성 확대

### 2. 문제 인식
- 마케팅 투자로 외형적 성장에도 경쟁 심화 영향으로 손익 개선 제한적
- 구매 전환율 향상을 통한 외적/질적 동시 성장 추구 필요
- 쇼핑몰 매출 = 유입 고객수(-마케팅 비용) X 클릭율(-마케팅 비용) X 구매 전환율(+매출/손익) X 판매가격(-프로모션/할인경쟁)
- 아마존의 컨텍스트 쇼핑 전략(Context Shopping)
  - 고객의 기호와 프로파일을 분석해서 구매욕구가 생겼을 때 즉각적으로 이를 해소 -> 구매로 연결 -> 높은 구매 전환율 확보  
  ```
  1. 다양한 기기들의 활용(Connecting Anywhere) : 다양한 라이프스타일의 데이터 확보 가능
  2. 실시간 추천 기능(Real-Time recommendation) : 고객의 컨텍스트를 정확히 이해하고 추천의 정확성 높임
  3. 제로 에포트의 구현(zero-effort) : 구매 단계뿐만아니라 결제 배송까지 자동화
  ```

### 3. 지향점
- 구매로 도달하지 못하는 97% 방문객에게 추천상품 제안 -> 동일 트래픽 상에서 추가 매출 확보
- 고객 이탈 발생 상품 페이지 특성에 맞춰서 *각 고객별 맞춤 상품 추천 솔루션 제공*
  ```
  - 쇼핑몰 랜딩 즉시 이탈하는 방문자 60% -> **통계형, 고객그룹기반 상품 추천**   
  - 구매 검토중인 방문자 25% -> **함께 본 상품 추천**   
  - 구매 시도 방문자 12% -> **함께 구매한 상품 추천**   
  - 실 구매자 3%
  ```
### 4. 단계별 고객 접점에 대한 솔루션 활용
- 구매까지 각 단계별로 추천 솔루션을 제공, 상품 상세 페이지까지 트래픽 유도
- 장바구니 페이지에서 구매 전활율 증가, 크로스셀 유도하여 전체 매출 상승에 기여

    |메인 페이지|카테고리 페이지|검색 페이지|상품 상세 페이지|장바구니 페이지|구매 페이지|
    |--------|-----------|--------|-------------|----------|---------|
    |선호 확률이 높은 고객 맞춤 상품 안내|카테고리 별 추천 상품 유도|상품 검색 실패 시, 가장 유사한 대안 추천|현재 검색중인 제품과 연관 혹은 부속된 제품을 추천|구매 직전의 제품과 연관 혹은 부속 제품을 추천|다른 고객들이 함꼐 구매한 이력이 높은 상품 추천|
    | 클릭률 증가, 이탈율 감소 |||구매 전환율 증가, 판매 SKU 증가|클릭율 증가, 고객 만족도 증가|구매 전환율 증가, 고객 만족도 증가|
    
  ```
  클릭률(Click-through rate, CTR) : 특정 링크를 클릭한 사용자의 수를 이용해 특정 웹사이트의 온라인 광고 캠페인 성공 여부와 이메일 캠페인의 효율성을 측정하는 방법
  이탈률(Bounce rate) : 단일 페이지 세션 수를 총 세션수로 나눈 비율. 단일 페이지뷰 이후 상호작용을 하지 않고 이탈한 세션의 비율
  구매전환율 (CVR, Conversion Rate) : 광고를 클릭하여 사이트에 들어온 방문자가 실제 구매 활동회원가입, 구매, 장바구니담기 등 광고주가 원하는 특정 행위을 하는 비율. 
    - 구매전환율 = 전환수(구매횟수)/클릭수*100
  판매 SKU (Stock Keeping Unit) : 재고 보관 단위. 상품 관리, 재고 관리를 위한 최소 분류 단위
  ```

### 5. 차별점
1.  시간 기반 추천 알고리즘
  - 4가지 관점의 시간 데이터 분석
    - 판매 상품 조회시간
    - 스크롤 시간
    - 컨텐츠 조회시간
    - 쇼핑몰 방문시간   
    
  - 분석 데이터 기반으로 실시간 추천
    - 데이터 실시간 분석
    - 고객 맞춤 상품 추천
    - 상품 및 컨텐츠 클릭   

2.  다양한 알고리즘 제공(조합/최적화 용이)
: 경쟁사 대비 5배 이상의 상품 추천 알고리즘 제공하여 정교한 타겟팅
  - 실시간 개인 맞춤 상품 추천 : 가장 최근에 클릭한 상품 분석
  - 함께 본 상품 추천 : 유사한 사용자의 데이터 분석
  - 함께 구매한 상품 추천 : 고객이 구매한 상품에 대한 분석
  - 통계형 기반 상품 추천 : 성별, 연령, 관심사 등 고객데이터 기반 분석
  - 검색어 기반 상품 추천 : 검색 키워드, 이용패턴으로 분석
  - 고객 그룹 기반 상품 추천 : 유사성이 높은 고객그룹과 다른 고객그룹간의 차이점 분석   

3.  트래픽 관리 및 보안, 운영 안정성 확보
  - AWS 사용하여 각 고객사별로 완전히 분리된 서버 환경 제공   

4.  QA Process
  - 솔루션 수정, 고도화, 모니터링을 통해 내부/외부 QA 및 모니터링 프로세스 구축   

5.  빠른 설치, 실시간 성과 관리
  - 모듈화된 20개의 추천 알고리즘을 조합하여 기획부터 설치까지
  - 관리자 페이지에서 솔루션 성과를 실시간 데이터로 확인 가능(AB 테스트)   


### 6. 예시
- 관리자 페이지 예시
  - 대시보드
  - 인기 추천상품 페이지(구매전환율 Top100)
  - 구매 전환율 및 기여 매출액 실시간 관리   

### 상품 추천 솔루션
1. 협업필터링 - 아이템, 상품추천
  - 가정 : '같은 콘텐츠, 상품을 선호하는 사용자는 콘텐츠 취향이 비슷할 것이다'
  - 상품, 콘텐츠 구매자들의 이력, 취향, 연관성을 분석하여 추천 알고리즘 제작
    ```
    - 정의 : 본인과 가장 비슷한 사용자의 구매 아이템을 추천받는 알고리즘
    - 내용 : 상품 구매 데이터를 기반으로 사용자 간의 유사도를 계산, 본인과 가장 유사한 사용자들이 구매했지만 본인이 구매하지 않은 상품 추천
    - 활용방법 : 상품 추천, 유사 사용자들에대한 마케팅 전략 수립
    - 필요자료 및 데이터 : 사용자 아이디 별 구매상품, 장바구니에 담은 상품, 찜한 상품 등
    ```
  ex   
    - 콘텐츠 : 왓챠, 레진코믹스, 망고플레이트
    - 커머스 : 11번가, 티몬, 신세계
    - 해외 : 아마존, 페북, 넷플릭스   
    
2. 협업필터링 - 유저, 상품추천
  - 가정 : '사용자가 구매했던 상품들과 연관성있는 상품들을 선호할 것이다'
  - 사용자가 구매한 상품에 대한 유사도, 취향, 연관성을 분석해서 추천 알고리즘 제작
    ```
    - 정의 : 본인이 산 상품과 가장 비슷한 상품을 추천받는 알고리즘
    - 내용 : 상품 구매 데이터를 기반으로 상품간의 유사도를 계산, 본인이 구매한 상품과 가장 유사한 상품 추천
    - 활용방법 : 상품 추천, 유사상품 패키지로 묶어서 할인판매
    - 필요자료 및 데이터 : 사용자 아이디 별 구매상품, 장바구니에 담은 상품, 찜한 상품 등
    ```
  ex 동일   
     
3. 콘텐츠 기반 추천 - 상품 설명 기반
  - 상품, 콘텐츠의 상세정보를 기반으로 인기상품과 유사한 상품 추천(키워드 to 상품)
  - 인기/일반상품에 대한 상세정보를 기반으로 추천알고리즘 구성. 구매이력 없이 조회정보 혹은 상품 상제정보만으로 구동 가능
    ```
    - 정의 : 본인이 산 상품과 컨텐츠적으로 비슷한 상품들을 추천받는 알고리즘
    - 내용 : ex 고양이 영상 시청 -> 다른 고양이 영상 추천
    - 활용방법 : 상품 추천
    - 필요자료 및 데이터 : 상품정보, 내용, 설명 
    ```   
  
4. 연관 상품 - 장바구니 - 함께 본/구매한 상품 추천
  - 고객의 구매 의사결정을 도와줄 수 있는 대체재 및 보완재 상품 추천
  - 대체재 : 고객이 현재 보고있는 상품과 관련성이 높은 상품 추천 -> 구매전환 유도
  - 보완재 : 고객이 상품을 구매하려고 할 때 함께 구매하면 좋은 상품 추천 -> 추가 구매 유도(장바구니 페이지)   
   
5. 통계형 기반 상품추천
  - 인기 상품 외에 구매로 전환될 확률이 높은 상품추천 등 다양한 통계 데이터 기반의 상품추천
  - 종류 : 실시간 많이 본 상품 top, 구매전환율 top(사용자 뷰 대비 구매율 높은), 많이 본/팔린 상품 top, 추천 클릭률(추천리스트 노출 대비 클릭이 많이 발생) top , 추천 유입률(상품의 유입경로 중 추천유입률이 높은 상품) top   
   
6. 검색어 기반 상품추천
  - 검색키워드 + 고객 이용패턴분석하여 연관 상품 추천   
   
7. 고객 그룹기반 상품추천
  - 고객 행동 기반 데이터를 분석하여 고객들 간의 유사성/연관성 등의 변수를 설정, 변수간 상관계수와 유사도 등으로 분류 분석하여 추천
  - 성별, 연령, 관심사 등 고객 관련 통계 기반으로 특정 고객 그룹에게 가장 적합한 인기상품 추천
  ```
  - 정의 : 고객별 유사성을 측정하여 유사성이 높은 대상 집단을 분류하고 군집에 속한 고객들의 유사성과 서로다른 군집에 속한 고객들의 차이점 분석
  - 내용 : 고객별 세그멘트를 구분하여 관리(Heavy user, vvip/vip/normal user)
  - 활용방법 : 고객 계층별 마케팅 전략을 달리하고 관리를 달리하여 구매전환율 높이기
  - 필요자료 및 데이터 : 고객 추적 자료, 로그자료 등
  ```   
   
8. 고객 후기 기반 상품추천
  - 사용자들의 후기 정보를 분석, 후기 작성/조회/클릭한 유저간 유사도 계산하여 상품 추천
  ```
  - 정의 : 고객 후기 분석을 활용한 상품 추천
  - 내용 : 상품 및 서비스에 대한 리뷰/후기 등 분석하여 토픽별로 나눠서 보여주기
  - 활용방법 : 해당 후기에 관여된 유저간 유사도 분석을 통한 추천
  - 필요자료 및 데이터 : 유저들의 후기 정보
  ```   
       
10. 딥러닝 기반 상품추천
  - 딥러닝으로 무한히 많은 변수 예측에 활용
  ```
  - 정의 : 딥러닝 사용
  - 내용 : 단순 구매내역, 사용자 메타정보(성별, 나이, 거주지, 이용기기, 접속정보, 클릭정보, 스크롤, 조회시간, 검색어, 방문경로 등) 등 모든변수 활용
  - 활용방법 : 상품, 사용자가 무수히 많은 경우 사용. 유튜브의 추천 알고리즘 방식
  - 필요자료 및 데이터 : 사용자 메타정보, 로그정보.
    -> 연산시간이 오래걸릴 수 있음 -> 소량의 변수들로 먼저 필터링하여 후보 상품군을 추출 후 후보 상품군에서 딥러닝 사용
  ```
     
11. 유사 이미지 검색 기반 상품추천
  - 이미지, 영상 등의 특징을 추출, 분류 분석하여 유사 이미지, 영상을 찾거나 분류
  ```
  - 정의 : CNN, 딥러닝을 통해 이미지 영상 분석 후 유사 이미지/영상 탐색 및 분류에 활용
  - 내용 : 이미지를 RGB/pixel단위로 분류하여 유사 이미지/영상 탐색 및 분류에 활용
  - 활용방법 : 유사 상품 및 영상 탐색/추천
  - 필요자료 및 데이터 : 이미지, 영상에 대한 다량의 데이터로 사전 딥러닝 학습 필요
  ```
